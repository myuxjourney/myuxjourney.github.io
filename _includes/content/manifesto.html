Project Mercury

This is a rough bootstrap I use to get a simple, close to static project up and running. This setup can be applied to any codebase without much changes at all. The big question is why?

I wanted an extremely slim baseline with my preferred grid system setup, along with a small smattering of styling, other than that, thats it. Close to bare metal as possible.

Content Agnostic CSS

For the love of all that is good and holy, DO NOT write CSS classes based on the content. Inevitably when you do that you'll need to use it elsewhere with different content, which means either you dupe the code and rename, or refactor for a re-usable content-agnostic naming convention. Do it right the first time.

Simple Grid

If I see a class column one-third I get angry. That might make sense on the desktop, but probably not on a tablet, or definitely not on your phone. It makes things confusing and syntactically incorrect. By approaching the build with an approach of modules and facets we can write cleaner CSS without worrying about floating or clearfixes, no ugly code, and it makes sense on all screen sizes.
